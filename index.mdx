Oxichrome is a Rust framework for building browser extensions that compile to WebAssembly. Annotate your Rust code with proc macros like `#[oxichrome::popup]` and `#[oxichrome::background]`, and `cargo oxichrome build` handles everything — compilation, wasm-bindgen, manifest generation, and JS shim generation — producing a ready-to-load extension with zero hand-written JavaScript.

## Get up and running

<Steps>
  <Step title="Install the CLI">
    Install the `cargo-oxichrome` subcommand from crates.io.

    ```bash
    cargo install cargo-oxichrome
    ```
  </Step>
  <Step title="Scaffold a new extension">
    Create a new project with the correct structure and dependencies pre-configured.

    ```bash
    cargo oxichrome new my-extension
    cd my-extension
    ```
  </Step>
  <Step title="Write your extension in Rust">
    Use proc macros to define your background worker and popup UI — no JavaScript needed.

    ```rust src/lib.rs
    use leptos::prelude::*;
    use oxichrome::prelude::*;

    #[oxichrome::extension(
        name = "My Extension",
        version = "1.0.0",
        permissions = ["storage"]
    )]
    struct Extension;

    #[oxichrome::background]
    async fn start() {
        oxichrome::log!("Running!");
    }

    #[oxichrome::popup]
    fn Popup() -> impl IntoView {
        view! { <p>"Hello from Rust."</p> }
    }
    ```
  </Step>
  <Step title="Build and load">
    Build for Chrome, then load the output folder as an unpacked extension.

    ```bash
    cargo oxichrome build
    ```

    Load `dist/chromium/` in `chrome://extensions` with **Load unpacked**.
  </Step>
</Steps>

## Why Oxichrome?

<CardGroup cols={2}>
  <Card title="Zero JavaScript" icon="ban">
    Your entire extension — background workers, popup UI, event handlers — compiles to WebAssembly. You never write a `.js` file.
  </Card>
  <Card title="Type-safe Chrome APIs" icon="shield-check">
    Bindings to `chrome.storage`, `chrome.tabs`, and `chrome.runtime` are fully typed and async-ready with Rust's `Result` type.
  </Card>
  <Card title="Reactive UI with Leptos" icon="display">
    Build popup and options page UIs using Leptos — a fine-grained reactive framework with familiar JSX-like syntax in Rust.
  </Card>
  <Card title="Chrome & Firefox" icon="globe">
    A single codebase targets both Chromium browsers and Firefox via `--target firefox`, generating Manifest V3-compatible output for each.
  </Card>
</CardGroup>
