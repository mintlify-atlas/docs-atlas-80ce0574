Content scripts run inside web pages rather than in the extension's background context. Oxichrome lets you write them in Rust using the `#[oxichrome::content_script]` proc macro — the build pipeline generates the JavaScript shim and wires up the manifest entries automatically.

## The `content_script` macro

Apply `#[oxichrome::content_script]` to an `async fn`. The function becomes the entry point for your content script:

```rust
#[oxichrome::content_script(matches = ["<all_urls>"])]
async fn inject() {
    oxichrome::log!("Content script running on this page!");
}
```

The function **must be `async`**. The macro enforces this at compile time and will produce an error if you omit the `async` keyword.

## The `matches` parameter

`matches` is a required array of [URL match patterns](https://developer.chrome.com/docs/extensions/develop/concepts/match-patterns) that control which pages your content script is injected into.

```rust
// All pages
#[oxichrome::content_script(matches = ["<all_urls>"])]
async fn inject_everywhere() { /* ... */ }

// A specific domain and all subpaths
#[oxichrome::content_script(matches = ["https://example.com/*"])]
async fn inject_example() { /* ... */ }

// Multiple patterns
#[oxichrome::content_script(matches = ["https://*.github.com/*", "https://*.gitlab.com/*"])]
async fn inject_git_hosts() { /* ... */ }
```

## The `run_at` parameter

`run_at` controls when in the page lifecycle the script is injected. It accepts a variant of the `RunAt` enum:

| Value | Timing |
|---|---|
| `DocumentStart` | Before any DOM is constructed or scripts run |
| `DocumentEnd` | After the DOM is complete, before subresources (images, etc.) load |
| `DocumentIdle` | After the page is idle — default when `run_at` is omitted |

```rust
#[oxichrome::content_script(
    matches = ["https://example.com/*"],
    run_at = DocumentStart
)]
async fn early_inject() {
    oxichrome::log!("Injected before DOM construction.");
}
```

<Tip>
Omit `run_at` entirely to use `DocumentIdle`, which is the safest default for most use cases since the page is fully loaded and idle before your script runs.
</Tip>

## How content scripts run

Content scripts execute in the context of the web page, not the extension background. This means:

- You have access to the page's DOM
- You are subject to the same-origin policy of that page
- You cannot directly call `chrome.*` APIs that require background context (use message passing instead)

The generated JavaScript shim loads the WASM module via `chrome.runtime.getURL` and calls your exported function:

```javascript
// Auto-generated by oxichrome. Do not edit.
// Content script: inject
(async () => {
    try {
        const src = chrome.runtime.getURL('wasm/my_extension.js');
        const wasm = await import(src);
        await wasm.default();
        wasm.__oxichrome_cs_inject();
    } catch (e) {
        console.error('[oxichrome] Content script "inject" failed to initialize:', e);
    }
})();
```

## Generated output

For each `#[content_script]`-annotated function, Oxichrome generates:

- `content_script_{fn_name}.js` in `dist/{browser}/` — the JS loader shim
- A corresponding entry in `manifest.json` under `content_scripts`
- A `web_accessible_resources` entry so the WASM files are accessible from page context

For a function named `inject`, you get `content_script_inject.js`.

## The `all_frames` parameter

By default, content scripts run only in the top-level document frame. Set `all_frames = true` to inject into every frame on the page, including iframes:

```rust
#[oxichrome::content_script(
    matches = ["https://example.com/*"],
    all_frames = true
)]
async fn inject_all_frames() {
    oxichrome::log!("Running in every frame on this page.");
}
```

## The `css` parameter

Use the `css` parameter to inject CSS files from your `static/` directory alongside the content script. The files are listed in the manifest's `content_scripts.css` array and injected automatically by the browser — you do not need to load them from Rust:

```rust
#[oxichrome::content_script(
    matches = ["https://example.com/*"],
    css = ["injected.css"]
)]
async fn inject_with_styles() {
    oxichrome::log!("Running with injected styles.");
}
```

Place `injected.css` in your project's `static/` directory so it gets copied into the build output.

## Permissions

You do **not** need to add a `content_scripts` permission to the `permissions` array in your `#[extension]` macro. The URL match patterns on the macro itself are sufficient — Oxichrome writes them directly into the manifest's `content_scripts` block.

```rust
// No "content_scripts" permission needed here
#[oxichrome::extension(
    name = "My Extension",
    version = "1.0.0",
    permissions = ["storage"]
)]
struct MyExtension;
```

## Multiple content scripts

You can define multiple content script functions in the same crate. Each one generates its own JS file and manifest entry:

```rust
#[oxichrome::content_script(matches = ["https://example.com/*"])]
async fn inject_example() {
    oxichrome::log!("On example.com");
}

#[oxichrome::content_script(
    matches = ["https://*.github.com/*"],
    run_at = DocumentEnd
)]
async fn inject_github() {
    oxichrome::log!("On GitHub");
}
```
