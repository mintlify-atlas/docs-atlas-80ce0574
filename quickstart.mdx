Oxichrome gives you a single CLI command that takes a Rust project from source to a browser-loadable extension. This guide walks you through installing the toolchain, scaffolding a new project, writing the counter example, building it, and loading it in Chrome and Firefox.

<Steps>
  <Step title="Install the CLI">
    Install `cargo-oxichrome` from crates.io. This adds the `cargo oxichrome` subcommand to your Cargo installation.

    ```bash
    cargo install cargo-oxichrome
    ```

    The CLI also needs the `wasm32-unknown-unknown` target and `wasm-bindgen-cli`. Both are installed automatically the first time you run `cargo oxichrome build`, with version matching handled for you.

    <Note>
      Rust stable is required. Run `rustup update stable` if you haven't updated recently.
    </Note>
  </Step>

  <Step title="Scaffold a new project">
    Create a new extension project with the correct structure and dependencies pre-configured.

    ```bash
    cargo oxichrome new my-extension
    cd my-extension
    ```

    This creates a `Cargo.toml` with the required dependencies and a `src/lib.rs` with a minimal working extension. The generated `Cargo.toml` looks like this:

    ```toml Cargo.toml
    [package]
    name = "my-extension"
    version = "0.1.0"
    edition = "2021"

    [lib]
    crate-type = ["cdylib"]

    [dependencies]
    oxichrome = { version = "0.1" }
    wasm-bindgen = "0.2"
    serde = { version = "1", features = ["derive"] }
    ```
  </Step>

  <Step title="Write your extension">
    Replace `src/lib.rs` with the counter extension below. It demonstrates proc macros, `chrome.storage` bindings, a Leptos popup UI, and an options page — all in Rust.

    ```rust src/lib.rs
    use leptos::prelude::*;
    use oxichrome::prelude::*;

    #[oxichrome::extension(
        name = "Counter Extension",
        version = "0.1.0",
        description = "A simple counter stored in chrome.storage.local",
        permissions = ["storage"]
    )]
    struct CounterExtension;

    #[oxichrome::background]
    async fn start() {
        oxichrome::log!("Counter Extension background service worker started!");
    }

    #[oxichrome::on(runtime::on_installed)]
    async fn handle_install(details: oxichrome::__private::wasm_bindgen::JsValue) {
        oxichrome::log!("Counter Extension installed: {:?}", details);

        if let Err(e) = oxichrome::storage::set("counter", &0i32).await {
            oxichrome::log!("Failed to initialize counter: {}", e);
        } else {
            oxichrome::log!("Counter initialized to 0.");
        }
    }

    #[oxichrome::popup]
    fn Popup() -> impl IntoView {
        let count = RwSignal::new(0i32);

        Effect::new(move || {
            wasm_bindgen_futures::spawn_local(async move {
                if let Ok(Some(val)) = oxichrome::storage::get::<i32>("counter").await {
                    count.set(val);
                }
            });
        });

        let update = move |delta: i32| {
            count.update(|c| *c += delta);
            let val = count.get_untracked();
            wasm_bindgen_futures::spawn_local(async move {
                let _ = oxichrome::storage::set("counter", &val).await;
            });
        };

        let increment = move |_| update(1);
        let decrement = move |_| update(-1);
        let reset = move |_| {
            count.set(0);
            wasm_bindgen_futures::spawn_local(async move {
                let _ = oxichrome::storage::set("counter", &0i32).await;
            });
        };

        view! {
            <h1>"Counter"</h1>
            <div>{move || count.get()}</div>
            <div>
                <button on:click=decrement>"-"</button>
                <button on:click=increment>"+"</button>
                <button on:click=reset>"Reset"</button>
            </div>
        }
    }

    #[oxichrome::options_page]
    fn Options() -> impl IntoView {
        view! {
            <h1>"Counter Extension Settings"</h1>
        }
    }
    ```

    A few things to notice:

    - `#[oxichrome::extension]` declares the extension metadata that goes into `manifest.json`.
    - `#[oxichrome::background]` marks the async function that runs as the service worker.
    - `#[oxichrome::on(runtime::on_installed)]` registers a Chrome event listener.
    - `#[oxichrome::popup]` marks a Leptos component as the popup UI.
    - `oxichrome::storage::get` and `oxichrome::storage::set` are async and return `Result`.
  </Step>

  <Step title="Build for Chrome">
    Run the build command from inside your project directory.

    ```bash
    cargo oxichrome build
    ```

    This compiles your Rust to WASM, runs wasm-bindgen, generates `manifest.json` and JS/HTML shims, and writes everything to `dist/chromium/`.

    <Tip>
      Run `cargo oxichrome build --release` for an optimized build with smaller WASM output.
    </Tip>
  </Step>

  <Step title="Load in Chrome">
    1. Open Chrome and navigate to `chrome://extensions`.
    2. Enable **Developer mode** using the toggle in the top-right corner.
    3. Click **Load unpacked**.
    4. Select the `dist/chromium/` folder from your project.

    Your extension is now active. Click the extension icon in the toolbar to open the popup and try the counter.
  </Step>

  <Step title="Build for Firefox">
    To target Firefox, pass the `--target firefox` flag.

    ```bash
    cargo oxichrome build --target firefox
    ```

    Output goes to `dist/firefox/` with a Firefox-compatible Manifest V3 `manifest.json` (including the `gecko` browser-specific settings).

    To load it in Firefox:

    1. Open Firefox and navigate to `about:debugging#/runtime/this-firefox`.
    2. Click **Load Temporary Add-on**.
    3. Select `dist/firefox/manifest.json`.

    <Note>
      Temporary add-ons in Firefox are removed when you close the browser. For persistent installation during development, use a [signed extension](https://extensionworkshop.com/documentation/publish/signing-and-distribution-overview/) or Firefox Developer Edition.
    </Note>
  </Step>
</Steps>
